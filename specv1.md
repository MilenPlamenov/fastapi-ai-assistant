# AI Assistant API - Project Specification

## Overview

Build a FastAPI-based AI Assistant API that uses a locally running Ollama model for inference.

The goal is to learn FastAPI while creating a realistic AI backend application with clean architecture, validation, API documentation, and separation of concerns.

The application must expose REST endpoints that allow users to send prompts and receive responses generated by Ollama.

---

## Tech Stack

* Python 3.12+
* FastAPI
* Uvicorn
* Pydantic
* HTTPX
* Ollama
* SQLite (prepare for future persistence)
* VS Code + WSL development environment

## Functional Requirements

### Health Check Endpoint

Route:

```http
GET /
```

Response:

```json
{
  "message": "AI Assistant API is running"
}
```

---

### Chat Endpoint

Route:

```http
POST /chat
```

Request:

```json
{
  "message": "Explain FastAPI"
}
```

Response:

```json
{
  "response": "FastAPI is..."
}
```

The endpoint must:

1. Validate input using Pydantic.
2. Call the Ollama service.
3. Return the generated response.

---

## Ollama Integration

Create a dedicated service layer.

File:

```text
services/ollama_service.py
```

Use HTTPX AsyncClient.

Connect to:

```text
http://localhost:11434/api/generate
```

Default model:

```text
llama3
```

Request payload:

```json
{
  "model": "llama3",
  "prompt": "<user_prompt>",
  "stream": false
}
```

Return only the generated text.

Handle connection errors gracefully.

## Validation

Use Pydantic models.

ChatRequest:

```python
message: str
```

Validation rules:

* required
* minimum length: 1
* maximum length: 4000

ChatResponse:

```python
response: str
```

---

## Error Handling

Return proper HTTP status codes.

Examples:

400 - invalid request

503 - Ollama unavailable

500 - unexpected server error

Use FastAPI exception handlers where appropriate.

---

## Async First

All routes and service calls should be asynchronous.

Use:

```python
async def
```

throughout the application.

---

## Environment Configuration

Create:

```text
.env
```

Variables:

```env
OLLAMA_URL=http://localhost:11434
OLLAMA_MODEL=llama3
```

Load configuration from environment variables.

Do not hardcode configuration values.
